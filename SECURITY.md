# Security Policy

## Supported Versions

Security fixes are provided for:

- The latest release
- The `main` branch

Older release lines may not receive security patches.

## Reporting a Vulnerability

Please do **not** report vulnerabilities in public issues.

Use one of the following private channels:

1. GitHub Security Advisory (preferred):
   - Repository -> Security -> Advisories -> Report a vulnerability
2. Direct maintainer contact on GitHub, if the advisory flow is unavailable

## What to Include

Please include as much of the following as possible:

- Affected version, commit hash, or tag
- Technical impact and attack scenario
- Reproduction steps / proof of concept
- Suggested mitigation (if available)

## Response Process

The project will:

1. Acknowledge receipt as soon as possible
2. Validate and triage severity
3. Work on a fix and coordinate responsible disclosure
4. Publish remediation guidance once a patch is available

## Disclosure

We follow responsible disclosure. Please avoid public disclosure until a fix is published.
